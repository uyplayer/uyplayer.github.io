简单的谈一下过拟合、欠拟合问题

uyplayer
Machine Learning

发布于：2020年4月2日
次浏览

字数：845字

时长：2分钟

过度拟合仅意味着学习模型过于依赖训练数据，而欠拟合意味着该模型与训练数据的关系较差。 理想情况下，这两者都不应该存在于模型中，但是通常很难消除它们。简单的说过度拟合是过度训练模型的结果，拟合不足是模型过于简单的结果，两者都导致高泛化误差。

使用数据集来预测或分类问题时，我们首先通过将设计模型应用于训练集，然后应用于测试集来检测准确性。 如果准确性令人满意，我们可以在机器学习模型中增加或减少数据特征，或者选择特征工程或通过应用特征工程来提高数据集预测的准确性。 但是有时候我们的模型给出的结果很差。 为什么会这样？ 我们如何处理呢？ 可以通过对“过度拟合”和“欠拟合”的概念进行简要研究来解释所有这些问题。

过度拟

过度拟合是指模型在原始数据上估计您要建模的变量确实很好，但在新数据集（保留，交叉验证，预测等）上估计得不好。 您的模型中变量或估计量过多（虚拟变量等），这会使您的模型对原始数据中的噪声变得过于敏感。 由于过度拟合原始数据中的噪声，因此模型的预测效果很差。

怎么解决过度拟合？

  • 交叉验证：查找样本外预测误差的标准方法是使用5倍交叉验证。
  • 修剪：在建立相关模型时，修剪被广泛使用。
  • 它只是删除了对问题几乎没有预测能力的节点。
  • 正则化：它引入了一个成本项，目的是引入具有目标功能的更多Feature。 因此，它试图将许多变量的系数推为零，从而降低成本项。
  • 为了避免过度拟合，如果有很多Feature，则添加正则化。 正则化迫使参数的大小变小（缩小了假设空间）。 为此，在成本函数中添加一个新Feature

欠拟合

欠拟合是指模型无法很好地估计原始数据或新数据中的变量。 您的模型缺少一些变量，这些变量对于更好地估计和预测因变量的行为是必需的。

怎么解决欠拟合？

  • 更重要的是更多数据无济于事。 实际上，这可能会增加训练误差。 因此，我们应该增加更多Feature。 因此，扩大了假设空间。 这包括从现有功能中制作新的Feature。 同样，更多参数也可以扩展假设空间。

总结：

  • 过拟合：过分依赖训练数据
  • 欠拟合：未能学习训练数据中的关系
  • 高方差：模型根据训练数据显着变化
  • 高偏差：对模型的假设不够导致忽略训练数据
  • 过拟合和欠拟合导致测试集的泛化性差
